    Bug Reporting

    Bug reporting is an important part of development for any project because it ensures a better, more accurate user experience. I do my best to test new features for proper functionality before officially adding them to MOES, but I can't test every feature on every game on every console for bugs by myself as that would take thousands of years. Your bug reports are greatly appreciated.

    What is a Bug?

    A bug in an emulator is the failure to properly execute game code and/or emulate intended game behavior. Bugs can create all sorts of bizarre behavior in games including, but not limited to:

    • Incorrect Graphics
    • Incorrect Audio
    • Incorrect Game Behavior
    • Games Freezing Up
    • Games Failing to Run

    How to Report a Bug

    There are two ways to report a bug. The first, and preferred method is by filing an issue to the MOES GitHub page. If you do not know what github is or how to file an issue there, the other way to file a bug report is to email me. Please make sure to include all of the information listed below.

    In your Bug Report

    Please Include

    • Which console you were using
    • Which ROM you were playing
    • A brief description of the bug
    • Relevant messages logged to the console if any (hit f12 to display console)
